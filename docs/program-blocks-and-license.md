# 프로그램 블록 이해 & 실습 기록 (2026-06-16)

> 대상: PLC 비전문가(웹/IT 개발자) 눈높이. 작성: Claude.
> 교재 프로젝트: `A2_전착오븐_PANELTEST` (실제 BSFA 프로젝트, GUI 제한 모드로 열람)

---

## 1. 프로그램 블록이란?

**"Program blocks"는 폴더다.** 그 안에 PLC의 실제 동작 프로그램이 **블록** 단위로 들어 있다. 블록은 4종류:

| 블록 | 웹/IT 비유 | 역할 |
|------|-----------|------|
| **OB** | `main()` / 이벤트 핸들러 | 시스템이 자동 호출 (프로그램 시작점). `OB1`은 무한 루프처럼 계속 돈다 |
| **FC** | 순수 함수 (stateless) | 입력 → 출력, **기억 없음** |
| **FB** | 클래스 인스턴스 (stateful) | **상태를 기억** (타이머·모터 제어 등) |
| **DB** | 데이터 객체 / JSON 설정 | **값만 저장**, 로직 없음 |

- FB는 상태를 기억해야 하므로 **자기 전용 데이터블록(DB)을 하나 달고 다닌다.** ← DB가 등장하는 이유.

## 2. 태그 vs 블록 vs 데이터블록 (제일 헷갈리는 부분)

- **태그** = 신호 **이름표** (낱개 변수). 블록과 **별개의 딴 동네**.
- **프로그램 블록** = **폴더** (블록들을 담는 곳).
- **데이터블록(DB)** = 그 폴더 안의 **한 종류 블록** (값 저장용).
- 태그(낱개 변수)와 DB(값 묶음 객체)가 닮아 보이지만 → **변수 한 개 vs 구조체 하나**의 차이.

> 외울 한 문장: **태그 ≠ 블록.** 그리고 **데이터블록은 'Program blocks 폴더' 안의 한 종류.**

## 3. 실제 PANELTEST 구조 (오늘 화면으로 확인)

```
PLC 1대: A2_CC01_CPU [CPU 1517F-3]   (모델명 F = 안전(Failsafe) CPU)
├─ 📋 PLC tags (태그) — 폴더별 정리: Default[725], 01_Timer[254],
│                        00_Gp, 01_InOut(CC01[88], DP01[208], OP01[104]...), ...
└─ 📦 Program blocks (동작 프로그램) — 기능별 폴더:
     ├ 01_공통 / 02_제어 / 03_장비 / 04_인터록 / 05_Safety / 09_PCR / System blocks
     └ 예) 02_제어 = FC_Global [FC300]  +  하위폴더(01_CC01, 02_DP, 03_OP, 04_WOP_DBs)
```

→ 태그도, 블록도 **기능별 폴더로 체계적으로 정리**돼 있음. 분석·자동화하기 좋은 깔끔한 프로젝트.

## 4. ⚠️ 라이선스 경계 (오늘의 핵심 발견)

블록(`FC_Global`)을 열려고 하자 곧바로 오류 대화상자:

> **Automation License Manager — "License 'STEP 7 Professional' was not found."**

단, 이 오류는 **편집·컴파일·다운로드 같은 기능만 제한**한다는 경고다. **블록 열람(보기)은 제한 모드로 정상 작동**한다 — OK/Esc로 오류창을 닫으면 래더·SCL 내용을 볼 수 있다. (※ 초기에 "블록 열기 불가"로 잘못 적었다가, 실제 열람되는 것을 확인하고 **정정함** — 2026-06-16. 상세: [tia-gui-troubleshooting.md](tia-gui-troubleshooting.md))

| 작업 | 필요 라이선스 | 이 PC(라이선스 없음) |
|------|------------|-----------------|
| 태그 보기 / XML 내보내기 | 불필요 | ✅ 됨 |
| IO 리스트(CSV) → 태그 XML 변환 | 불필요 (순수 텍스트 변환) | ✅ 됨 (`tools/io-convert` 완성) |
| 블록 **열람**(래더/SCL 보기) | **불필요 (제한 모드)** | ✅ 됨 (내용 확인 가능) |
| 블록 **편집·컴파일·다운로드** | **STEP 7 Professional** | ❌ 막힘 |
| 태그/블록 import (프로젝트에 반영) | (라이선스 필요 추정) | ❌ 막힘 |

**입력 방식 메모(이 PC 환경):** TIA 화면 자동 조작 시 **마우스 단일 좌클릭**과 **저수준 키 입력(`keybd_event`)**만 안정적으로 작동. 더블클릭 / 우클릭 컨텍스트 메뉴 / `SendKeys`는 이 환경에서 불안정 → 폴더 펼치기는 토글(▶) 단일클릭, 키 입력은 `keybd_event` 사용.

## 5. 그래서 자동화 전략

- **태그 / IO 자동화 = 1순위** — 라이선스 없이 **지금 당장** 가능. (이미 `Convert-IoListToTags.ps1` 완성)
- **블록 관련(SCL 소스 생성·래더 분석·import)** — **STEP 7 Professional 라이선스 PC에서** 진행.
