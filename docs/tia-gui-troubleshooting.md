# TIA GUI 자동 조작 — 병목 & 해결 로그

> Claude가 TIA Portal 화면을 자동 조작(`tools/desktop/Ui.ps1`)할 때 만난 막힘·착오·해결책을 누적 기록한다.
> **목적: 동일한 실수를 반복하지 않기 위함.** 새 병목/해결을 만나면 여기에 계속 추가할 것.

---

## ✅ 작동하는 것 (검증됨)

- **마우스 단일 좌클릭** — 항목 선택, 폴더 토글(▶) 펼치기. 가장 안정적.
- **`keybd_event` 저수준 키 입력** — `Enter`(0x0D), 방향키(`↑`0x26 `↓`0x28 `→`0x27 `←`0x25), `Esc`(0x1B). **`SendKeys`보다 안정적.** 트리 네비게이션(↑↓), 폴더 펼침(→), 블록 열기(Enter), 대화상자 취소(Esc)에 사용.
  - 호출: `[WinUI]::keybd_event([byte]0xNN,[byte]0,[uint32]0,0)` (down) / `...[uint32]2,0)` (up).
  - **`Ui.ps1` 헬퍼(2026-06-16 추가):** `Ui-Enter` · `Ui-Down N` · `Ui-Up N` · `Ui-Expand` · `Ui-Collapse` · `Ui-Esc` · `Ui-FocusClick X Y`(키 입력 전 포커스 확보). **트리 네비게이션은 이 헬퍼를 쓸 것** — 좌표 무관이라 다른 PC에서도 그대로 동작.
- **블록 열람(보기)** — 라이선스 없이 **제한 모드로 가능** (래더/SCL 내용 확인 OK).

## ❌ 막히는 것 (이 환경)

- **더블클릭(`DblClick`)** — 블록/폴더 열기 안 됨 (선택만 되거나 무반응).
- **우클릭 컨텍스트 메뉴(`RClick`)** — 메뉴가 안 뜸.
- **`SendKeys`** 방향키/문자 — TIA에 전달 안 됨. (한글 입력은 원래도 클립보드 사용)

## ⚠️ 함정 / 주의

1. **좌표 클릭이 행 단위로 빗나감** → 클릭 전 반드시 **격자 캡처(`Save-Crop -Grid`)로 실제 좌표 재확인.** `Move`(hover)로 어느 행에 닿는지 검증 가능(행 하이라이트).
2. **`Ui-Activate` 후 `keybd_event Enter`가 트리 포커스를 잃고 엉뚱한 버튼('새 프로젝트' 등)을 누름** → 키 입력 전 **대상 항목을 단일 좌클릭으로 선택**해 트리에 포커스를 확실히 준 뒤 키 입력. (가능하면 Activate 직후 키 입력은 피하고, 클릭→키 순서로.)
3. **라이선스 오류 대화상자 ≠ 기능 완전 차단.** `"License 'STEP 7 Professional' was not found"`가 떠도 **OK/Esc로 닫으면 블록 열람은 됨.** 오류만 보고 "불가"로 성급히 결론짓지 말 것 — 닫은 뒤 중앙 에디터를 반드시 확인.
